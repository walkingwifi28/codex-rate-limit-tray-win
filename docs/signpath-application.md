# SignPath Foundation Application Notes

This document collects the information needed to apply for free open-source code
signing through SignPath Foundation.

## Project Summary

Codex Rate Limit Tray is a Windows task tray application for displaying
Codex/ChatGPT `wham/usage` rate-limit usage, remaining percentage, and reset
time. It is implemented as a .NET 8 WinForms application.

- Repository: https://github.com/walkingwifi28/codex-rate-limit-tray-win
- License: MIT
- Release artifacts: Windows x64 Inno Setup installer
- Package identifier: `WalkingWiFi.CodexRateLimitTray`
- Maintainer: repository owner `walkingwifi28`

## Why Signing Is Needed

The project distributes a Windows installer through GitHub Releases and intends
to reference that installer from Windows Package Manager manifests. Windows
Defender SmartScreen currently shows an unrecognized application warning for
unsigned installers. Signing is needed so users can verify that the installer was
produced by the open-source project and was not modified after release.

## Requested Signing Scope

The project requests signing for release installers only:

```text
CodexRateLimitTray-<version>-win-x64-setup.exe
```

The installer is generated by Inno Setup from the .NET publish output.

The project does not request signing for:

- Local developer builds
- Pull request artifacts
- Debug builds
- Third-party binaries
- Modified upstream software

## Build and Release Process

Releases are created from Git tags matching `vX.Y.Z`.

The GitHub Actions release workflow:

1. Checks out the tagged source.
2. Installs .NET 8.
3. Runs `dotnet test CodexRateLimitTray.sln --configuration Release`.
4. Publishes `src/CodexRateLimitTray/CodexRateLimitTray.csproj` for
   `win-x64`.
5. Installs Inno Setup.
6. Builds `installer/CodexRateLimitTray.iss`.
7. Uploads the unsigned installer as a GitHub Actions artifact.
8. Submits that artifact to SignPath.
9. Publishes only the signed installer and SHA256 checksum to GitHub Releases.

Unsigned release installers are not published.

```text
Free code signing provided by SignPath.io, certificate by SignPath Foundation.
```

The policy also documents the signed artifact scope, release rules, provenance,
and maintainer responsibilities.

## SignPath Configuration To Create After Approval

Suggested values:

- Project slug: `codex-rate-limit-tray-win`
- Signing policy slug: `release-signing`
- Artifact configuration slug: `inno-setup-installer`

GitHub repository variables expected by `.github/workflows/release.yml`:

- `SIGNPATH_ORGANIZATION_ID`
- `SIGNPATH_PROJECT_SLUG`
- `SIGNPATH_SIGNING_POLICY_SLUG`
- `SIGNPATH_ARTIFACT_CONFIGURATION_SLUG`

GitHub repository secret expected by `.github/workflows/release.yml`:

- `SIGNPATH_API_TOKEN`

The exact values should be copied from the SignPath project after the Foundation
application is approved.
